## 구현 단계 (디자인 에셋이 준비된 상태에서 진행)

### Step 0: 환경 세팅

| 순서 | 작업 | 상태 |
|---|---|---|
| S-0-1 | Node.js 설치 확인 | ⬜ |
| S-0-2 | Vite + TypeScript + Phaser 3 프로젝트 초기화 | ⬜ |
| S-0-3 | CLAUDE.md 작성 (게임 규칙, 코딩 컨벤션, 기술 스택) | ⬜ |
| S-0-4 | Git 초기화 | ⬜ |

### Step 1: 핵심 게임플레이 (Phase 1~5)

| Phase | 작업 | 완료 기준 | 상태 |
|---|---|---|---|
| P-1 | 그리드 렌더링 + 타일 생성 (초기 매칭 방지) | 8x8 보드가 실제 에셋으로 화면에 표시됨 | ⬜ |
| P-2 | 스왑 인터랙션 (터치/마우스) + 스왑 애니메이션 | 인접 타일 교환 가능 | ⬜ |
| P-3 | 매칭 감지 알고리즘 (3개 이상 수평/수직) | 매칭 자동 감지 | ⬜ |
| P-4 | 매칭된 타일 제거 + 중력 낙하 + 새 타일 채우기 | 빈 자리 자연스럽게 채워짐 | ⬜ |
| P-5 | 연쇄 반응 (cascade) 처리 | 낙하 후 추가 매칭 자동 처리 | ⬜ |

> Phase 5 완료 = 플레이 가능한 프로토타입

### Step 2: 게임성 완성 (Phase 6~8)

| Phase | 작업 | 상태 |

| P-8 | 레벨 시스템 + 목표 조건 (점수 달성, 특정 젬 N개 제거 등) | ⬜ |
