# 화면 규격 및 그리드 레이아웃 (D-2)

## 1. 기준 해상도

| 항목 | 값 |
|---|---|
| 기준 해상도 | **1080 x 1920 px** (9:16) |
| 방향 | **세로 (Portrait)** 고정 |
| 스케일링 | 가로 기준 fit → 세로 남는 영역은 배경 확장 |
| DPI 기준 | @2x (논리 해상도 540 x 960) |

### 지원 비율별 대응

| 비율 | 대표 기기 | 대응 방식 |
|---|---|---|
| 9:16 | 대부분 Android | 기준 해상도 그대로 |
| 9:19.5 | iPhone 14/15, Galaxy S24 | 상하 여백에 배경 확장 |
| 9:20 | iPhone 15 Pro Max | 상하 여백에 배경 확장 |
| 3:4 | iPad (태블릿) | 좌우 여백에 배경 확장 |

## 2. 안전 영역 (Safe Area)

```
┌──────────────────────────────┐
│      상단 안전 여백 (64px)     │  ← 노치/Dynamic Island/상태바
├──────────────────────────────┤
│                              │
│                              │
│       게임 콘텐츠 영역         │
│     (1080 x 1792 px)        │
│                              │
│                              │
├──────────────────────────────┤
│      하단 안전 여백 (64px)     │  ← 홈 인디케이터/제스처바
└──────────────────────────────┘
```

| 영역 | 크기 |
|---|---|
| 상단 안전 여백 | 64 px (상태바 + 노치 대응) |
| 하단 안전 여백 | 64 px (홈 인디케이터 대응) |
| 좌우 안전 여백 | 각 32 px |
| **사용 가능 영역** | **1016 x 1792 px** |

## 3. 게임 플레이 화면 레이아웃 (상세)

### 전체 배치 (1080 x 1920 기준)

```
Y 좌표 기준 (위에서 아래로)
═══════════════════════════════════════

[0 ~ 64]        상단 안전 여백
                 (투명 or 배경색)

[64 ~ 264]      상단 HUD (200px)
                 ┌─────────────────────────┐
                 │  목표 아이콘  │ 남은이동수 │
                 │  진행 바                  │
                 │  점수                     │
                 └─────────────────────────┘

[264 ~ 304]     HUD-보드 간격 (40px)

[304 ~ 1384]    게임 보드 (1080px)
                 ┌─────────────────────────┐
                 │                         │
                 │   8 x 8 타일 그리드      │
                 │   (1080 x 1080 px)      │
                 │                         │
                 └─────────────────────────┘

[1384 ~ 1424]   보드-하단 간격 (40px)

[1424 ~ 1624]   하단 부스터 바 (200px)
                 ┌─────────────────────────┐
                 │ [뒤집개] [믹서] [일시정지]│
                 └─────────────────────────┘

[1624 ~ 1856]   하단 여유 공간 (232px)
                 (배경 또는 장식)

[1856 ~ 1920]   하단 안전 여백 (64px)
```

## 4. 게임 보드 상세 규격

### 4.1 보드 크기

| 항목 | 값 |
|---|---|
| 보드 영역 | **1080 x 1080 px** (정사각형) |
| 보드 패딩 (좌우) | 각 28 px |
| 실제 타일 영역 | **1024 x 1024 px** |
| 그리드 | 8행 x 8열 |

### 4.2 타일 규격

| 항목 | 값 |
|---|---|
| 타일 크기 (렌더링) | **120 x 120 px** |
| 타일 간격 (gap) | **8 px** |
| 타일 + 간격 | 128 px |
| 보드 총 너비 | 120×8 + 8×7 = **1016 px** (좌우 4px 여유) |
| 타일 에셋 원본 | **256 x 256 px** (다운스케일링으로 선명도 확보) |
| 타일 터치 영역 | **128 x 128 px** (타일보다 약간 크게, 오탈 방지) |

### 4.3 보드 좌표 계산

```
타일[row][col]의 중심 좌표:
  x = 보드시작X + (col × 128) + 60
  y = 보드시작Y + (row × 128) + 60

보드시작X = (1080 - 1016) / 2 = 32
보드시작Y = 304
```

## 7. 애니메이션 타이밍 기준

| 타일 스왑 | 200ms | ease-in-out |

## 8. Stitch 디자인 입력용 요약

Stitch에 전달할 핵심 수치:

```
- 캔버스: 1080 x 1920 (세로)
- 배경색: #FFF8F0
- 보드 배경: #F5E6D3, 둥근 모서리(radius 32px)
- 보드 위치: 화면 중앙, 상단에서 304px
- 보드 크기: 1024 x 1024
- 타일: 120 x 120, 간격 8px, 둥근 모서리(radius 20px)
- HUD: 상단 64~264px 영역
- 부스터 바: 하단 1424~1624px 영역
- 폰트: Nunito (둥근 산세리프)
- 컬러: 파스텔 톤 (#FF9BAA, #C4A7E7, #A8E6CF, #FFD07B, #89CFF0, #A0725B)
- 전체 분위기: 미니멀, 코지, 따뜻한 베이커리
```
